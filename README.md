# 煜昌电器改善提案系统

本项目是煜昌电器改善提案、积分激励、礼品兑换与统计报表的一体化系统，正式交付形态为微信小程序。

## 功能范围

- 登录与角色权限：普通员工、部门评估组、财务复核、精益办复审、超级管理员
- 改善提案：普通改善提案、有效焦点课题、草稿、提交、部门评估、财务复核、精益办复审、评审委员会核准、驳回、无效关闭、归档
- 附件管理：提案附件与财务效益核算表上传、下载
- 积分账户：积分余额、流水、人工奖励、人工扣减
- 兑换管理：礼品库存、库存预占、兑换审核、发放确认
- 数据导入：员工、部门、礼品 CSV / XLS / XLSX 模板下载与导入
- 报表与日志：提案统计、积分排行、操作日志

## 本地运行

```bash
npm run dev
```

访问：

```text
http://localhost:3000
```

## 微信小程序开发

已新增原生微信小程序工程，目录为 `miniprogram/`，项目配置为 `project.config.json`。

使用方式：

1. 先运行后端：`npm run dev`
2. 使用微信开发者工具导入本项目根目录
3. 本地开发时已在 `project.config.json` 关闭 URL 校验
4. 当前开发环境请求地址为 `http://127.0.0.1:3000`
5. 登录账号为工号，初始密码为工号后加 `@`

当前小程序端已覆盖：

- 登录与角色鉴权
- 工作台
- 改善提案新建、草稿、编辑、删除、审批流
- 附件上传
- 积分账户与年度清零接口
- 礼品兑换、审核和发放
- 员工导入
- 管理中心、员工列表、报表、日志
- 修改密码

## 数据说明

- 本地运行数据保存在 `data/db.json`，该文件不会提交到 Git。
- 上传附件保存在 `data/uploads/`，该目录不会提交到 Git。
- 如需重置演示数据，停止服务后删除 `data/db.json`，再重新启动即可。
- 员工导入兼容字段：`姓名`、`工号`、`1级部门`、`2级部门`、`部门主管`、`手机号`、`职位`、`员工状态`、`性别`。
